# Decision Science Code

Open computational companion repository for the Decision Science knowledge series.

This repository supports articles on decision science, decision theory, uncertainty, expected value, expected utility, decision trees, Bayesian decision-making, risk analysis, sensitivity analysis, multi-criteria decision analysis, robust decision making, behavioral decision theory, complex systems, public policy, sustainability, healthcare, finance, organizational strategy, AI-assisted decision support, and decision governance.

## Repository Structure

- `articles/decision-science-judgment-under-uncertainty/`
  - `python/` — expected value, decision trees, regret analysis, robustness, and scenario comparison
  - `r/` — MCDA, sensitivity analysis, trade-off profiles, and reproducible decision summaries
  - `julia/` — high-performance scenario and robustness examples
  - `sql/` — alternatives, criteria, assumptions, evidence, scenarios, model runs, and decision records
  - `rust/` — command-line decision diagnostics scaffold
  - `go/` — decision scoring and scenario utility scaffold
  - `cpp/` — efficient expected-value and regret examples
  - `fortran/` — numerical decision-model examples
  - `c/` — low-level expected-value utilities
  - `docs/` — article notes and modeling principles
  - `data/` — synthetic datasets
  - `outputs/` — generated outputs
  - `notebooks/` — notebook placeholders

## Modeling Principles

1. Define the decision before modeling options.
2. Distinguish outcome quality from decision-process quality.
3. Make alternatives explicit.
4. Represent uncertainty honestly.
5. Surface values and trade-offs.
6. Use sensitivity analysis to test assumption dependence.
7. Prefer robustness over fragile optimization under deep uncertainty.
8. Document decision records for accountability and learning.
9. Treat computational models as supports for judgment, not replacements for responsibility.
