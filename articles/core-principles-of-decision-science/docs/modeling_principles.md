# Modeling Principles

## Define the decision first

A model should support a defined choice. Without a clear decision, analysis can become technical activity without decision relevance.

## Represent uncertainty honestly

Use probabilities where defensible, ranges where estimates are uncertain, scenarios where futures are plural, and robustness diagnostics where deep uncertainty matters.

## Surface values and trade-offs

Criteria, weights, thresholds, and constraints embed value judgments. They should be visible and reviewable.

## Test fragility

Use sensitivity and scenario analysis to identify assumptions that change the recommendation.

## Preserve accountability

Decision records should document framing, alternatives, evidence, uncertainty, criteria, rationale, dissent, monitoring indicators, and review triggers.
