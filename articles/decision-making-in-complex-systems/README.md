# Decision-Making in Complex Systems

This companion directory supports the article **"Decision-Making in Complex Systems."**

The workflows operationalize complex-system decision-making as a reproducible decision-science problem involving interdependence, feedback loops, delayed effects, adaptive response, spillover pressure, threshold risk, robustness, scenario comparison, monitoring triggers, and decision-record scaffolding.

## Core Claim

In complex systems, decisions do not enter a stable environment. They change the system that later decisions must face. Decision quality therefore depends on system structure, feedback, adaptation, thresholds, monitoring, and revision—not only on the apparent best option at the time of choice.

## Modeling Principles

1. Define the decision and system boundary.
2. Map interdependence, feedback loops, delays, and constraints.
3. Evaluate direct, indirect, delayed, and systemic consequences.
4. Compare strategies across stable, delayed-feedback, shock, coordination-stress, and adaptive-resistance scenarios.
5. Track threshold risk and vulnerability conditions.
6. Anticipate adaptation, strategic response, and metric gaming.
7. Preserve flexibility through staged, modular, and reversible decisions.
8. Monitor system response after action.
9. Define trigger points for revision.
10. Preserve decision records before hindsight rewrites the rationale.

## Quick Start

From this article directory:

```bash
python3 python/run_all_complex_system_workflows.py
Rscript r/run_all_complex_system_workflows.R
```

Outputs are written to:

```text
outputs/tables/
outputs/figures/
outputs/decision_records/
```
