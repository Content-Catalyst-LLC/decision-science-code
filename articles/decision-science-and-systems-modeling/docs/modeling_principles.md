# Modeling Principles

## Start with the decision

A systems model should support a decision question, not become an abstract diagramming exercise.

## Make structure explicit

Stocks, flows, feedback loops, delays, thresholds, incentives, and boundaries should be visible and contestable.

## Represent decisions as interventions

Decision options should change model parameters, flows, rules, signals, incentives, capacities, or relationships.

## Treat outputs as hypotheses

Model outputs are structured claims under assumptions, not guaranteed predictions.

## Connect modeling to accountability

Model purpose, assumptions, limits, scenarios, outputs, and decision influence should be documented.
