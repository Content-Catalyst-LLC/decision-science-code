# Decision Science in Organizational Strategy

This companion directory supports the article **"Decision Science in Organizational Strategy."**

The workflows operationalize organizational strategy as a reproducible decision-science problem involving strategic option comparison, scenario evaluation, bounded rationality, robust strategy, capability fit, adaptability, governance feasibility, assumption drift, strategic review cycles, decision records, and cross-language implementation scaffolds.

## Core Claim

Organizational strategy is not only vision, planning, positioning, or competitive advantage. It is a decision system made of linked commitments: which markets to enter, which customers to serve, which capabilities to build, which risks to absorb, which opportunities to ignore, which trade-offs to defend, and which time horizons should govern present action.

## Modeling Principles

1. Treat strategy as a decision system, not only a plan document.
2. Distinguish risk, uncertainty, ambiguity, deep uncertainty, and strategic surprise.
3. Compare genuine strategic alternatives rather than ratifying a preferred path.
4. Evaluate expected value, downside robustness, adaptability, capability fit, governance feasibility, and reversibility.
5. Calibrate commitment size to uncertainty, reversibility, timing, and resource constraints.
6. Include behavioral risk: overconfidence, groupthink, confirmation bias, escalation, and status incentives.
7. Analyze system effects, feedback loops, delays, path dependence, and policy resistance.
8. Connect strategy to actual budgets, incentives, decision rights, and review authority.
9. Track assumption drift and trigger review before collapse becomes visible.
10. Preserve assumptions, alternatives, dissent, trade-offs, commitments, and revision triggers in decision records.
