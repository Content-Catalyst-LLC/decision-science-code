# Modeling Principles

## Expected value is a benchmark

Expected value summarizes probability-weighted outcomes. It is most appropriate when outcomes are linear, probabilities are credible, and the decision-maker is effectively risk-neutral.

## Expected utility adds valuation

Expected utility transforms outcomes through a utility function. This allows risk aversion, risk neutrality, or risk seeking to be represented explicitly.

## Utility functions are assumptions

Utility functions should be documented, justified, and tested. They represent valuation, not objective fact.

## Sensitivity matters

Probability assumptions, outcome magnitudes, and risk-aversion parameters should all be varied to test ranking stability.
