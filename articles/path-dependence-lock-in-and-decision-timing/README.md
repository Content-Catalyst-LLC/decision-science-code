# Path Dependence, Lock-In, and Decision Timing

This companion directory supports the article **"Path Dependence, Lock-In, and Decision Timing."**

The workflows operationalize path-dependent decision-making as a reproducible decision-science problem involving switching costs, lock-in risk, option value, institutional routines, network dependence, decision timing, scenario comparison, strategic windows, exit paths, and decision-record scaffolding.

## Core Claim

Some decisions become more consequential over time because they change the future choice set. Early investments, routines, standards, contracts, network dependence, institutional expectations, and stakeholder adaptation can create lock-in. Decision timing matters because acting, waiting, staging, piloting, or reversing can alter future switching costs and available options.

## Modeling Principles

1. Treat decisions as sequences, not isolated events.
2. Identify how prior commitments shape the present choice set.
3. Separate sunk costs from real future switching costs.
4. Track switching-cost accumulation across investment, network dependence, and institutional routine.
5. Evaluate option value before it decays.
6. Compare paths across stable, technology-shift, policy-change, demand-shock, and late-switch scenarios.
7. Use modularity, interoperability, staging, and exit rights to reduce harmful lock-in.
8. Identify strategic windows and critical junctures before they close.
9. Preserve timing logic, lock-in risks, and exit paths in a decision record.
10. Commit deliberately where commitment is useful and preserve flexibility where uncertainty remains high.
